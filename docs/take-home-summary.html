Caseware Senior AI Data - Take-Home

Resumen de la solución: pipeline local, RAG híbrido y servidor MCP para consultas sobre procurement e inventario con respuestas citadas.

Repositorio: github.com/leonardorhojas/caseware-procurement-kb

1. Qué pedía la prueba

Caseware solicitó una plataforma local que permita a un agente AI (vía MCP) consultar una base de conocimiento con:

  • Facturas, órdenes de compra, envíos, reportes de inventario y contratos
  • PDFs e imágenes (con extracción/OCR pragmática)
  • Respuestas fundamentadas con referencias a fuentes
  • Matching cross-document entre registros relacionados

Sin cloud, sin frontend, sin over-engineering: pipeline reproducible + MCP funcional en 3-5 horas.

2. Pasos para ejecutar la solución

  1. Setup - Python 3.11+, venv, pip install -e ., opcional .env con OPENAI_API_KEY.
  2. Ingest (offline) - python scripts/ingest.py escanea data/, extrae texto, genera chunks, embeddings e índice en artifacts/.
  3. MCP - Conectar caseware-kb en Cursor (config en .cursor/mcp.json). El servidor lee el índice ya procesado; no re-extrae PDFs en runtime.
  4. Consultas - Preguntas en lenguaje natural en Cursor; el agente invoca tools MCP y devuelve respuestas con citas.
cd CASEWARE
python3 -m venv .venv && source .venv/bin/activate
pip install -e .
cp .env.example .env   # opcional
python scripts/ingest.py
# Conectar MCP caseware-kb en Cursor y preguntar en el chat

Opcional brew install tesseract + re-ingest para indexar 5 facturas JPG vía OCR.

3. Arquitectura implementada

data/ PDFs + JPGs → extract.py → chunk.py → embed.py → store.py → artifacts/
artifacts/ → KnowledgeBase.load() → MCP tools → Cursor / agente AI

Stack

  • Python 3.11, PyMuPDF, pytesseract
  • Embeddings: OpenAI (fallback local MiniLM)
  • SQLite + numpy (vectores en memoria al cargar)
  • MCP SDK (FastMCP, stdio)

Artefactos generados

  • artifacts/kb.sqlite - metadata + chunks
  • artifacts/vectors.npy - embeddings
  • artifacts/manifest.json - chunk IDs, provider

4. Modelo de datos y retrieval

Estrategia de retrieval (3 capas)

CapaCuándoCómo
Routing estructuradoOrder ID, gaps, inventarioSQL en metadata (matching.py)
Búsqueda vectorialPreguntas semánticas (contrato, conceptos)Cosine similarity sobre embeddings
Fallback keywordSin hits vectorialesSQL LIKE (no BM25)

Mejora futura: BM25 + FTS5 + fusión RRF con vector search para tokens exactos (Clause 7, SKUs, order IDs en texto libre).

5. Tools MCP expuestas

ToolParámetrosRetorno
search_knowledge_base query, top_k, doc_type, order_id Chunks con score, snippet, file_path, page, chunk_id
get_order_documents order_id Documentos agrupados por tipo (invoice, PO, shipping, ...)
find_document_gaps (ninguno) Invoices sin PO, shipping sin invoice, matches completos
list_inventory_reports (ninguno) Periodos y paths de reportes
get_source_excerpt chunk_id Texto ampliado para citar en la respuesta

El MCP consume datos indexados; evita devolver PDFs enteros - solo snippets estructurados con referencias.

6. Preguntas del reto - cómo se resolvieron

Pregunta (PDF)EnfoqueTool / métodoResultado validado
Which documents support order 10687? Join por order_id get_order_documents Invoice + shipping order (2 docs). Sin PO.
Which invoices are missing a matching purchase order? Set difference invoice IDs vs PO IDs find_document_gaps 10436, 10687, 10839. Full match: 10248, 10382, 10442, 10603, 10711.
What purchase order supports this invoice? Lookup por order_id del invoice get_order_documents + gaps Si no hay PO en el dataset, gaps lo reporta explícitamente.
Does this shipment match the related invoice? Bundle + comparación de líneas/total en texto indexado get_order_documents + contenido chunks Order 10436: match completo, total 2,210.8 en ambos.
Summarize contract terms relevant to supply of goods RAG semántico sobre contrato TotalEnergies search_knowledge_base Cláusulas 7, 9, 13, 14 - delivery, warranty, acceptance, remedies.
Are there mismatches between invoices, POs, and shipping? Análisis de sets por categoría find_document_gaps Listas de gaps + 5 order IDs fully matched.
What inventory reports are available, and what period? Metadata period en filename list_inventory_reports 7 reportes: 2016-07 a 2018-01 (cobertura ~trimestral).
Find evidence related to vendor, order, or item Order: structured; vendor/item: vector + keyword get_order_documents / search_knowledge_base Parcial - sin extracción dedicada de campos vendor; búsqueda semántica sobre chunks.

9. Decisiones de diseño y limitaciones

  • Simple sobre complejo: numpy + SQLite en lugar de Chroma/Elastic; suficiente para ~300 chunks.
  • Ingest offline: MCP rápido al arrancar; reprocesar solo cuando cambia data/.
  • Extracción vs retrieval: campos clave (order_id, period) en ingest; resto vía chunks + embeddings.
  • Citas: chunk_id + file_path + page + snippet; get_source_excerpt para profundizar.
  • AI-assisted: desarrollo con Cursor; validación manual + smoke tests MCP.
